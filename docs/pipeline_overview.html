What the program-10 test taught us

Three findings from the first end-to-end run, each with the fix it points to.

✓ Now fixed

Wrong dataset

The first test ran on shipped demo data. The correct dataset is /Volumes/IF_PHAGE/Hepatocyte/Programs0617 (K=50 cNMF programs).

gene_loading_K50_0_5_top300.csv = Name, Score, RowID. Young & aged perturbation files = target_name, program_name, log2FC, p-value, adj_pval.

Correct program 10 = a zonated hepatocyte metabolism / detox program (Rhbg, Glul, Oat, Cyp2e1, Cyp1a2, Slc22a1, Slco1b2, Gstm2, Axin2, Pon1), repressed by lipogenic regulators (Dgat2, Scap, Mlxipl, Insig1), with larger effects in young than aged.
Self-inflicted

Too many conflicts — because we asked for them

The protocol explicitly told the agent to investigate "perturbation-direction consistency" and "competing / contradictory explanations" — 2 of its 4 questions.

It obediently returned 5 contradictions + 4 direction conflicts, some of them artifacts of mismatched regulators.

Fix: flag contradictions only if they surface — don't hunt for them.
Two causes

Empty areas

(a) Three integration bugs. STRING enrichment never ran (empty enrichment block); the report's per-module DOI/status merge didn't fire; the presentation lexicon is liver-metabolic only (empty tags for other biology).

(b) Effort spread thin. A heavy 4-question protocol + broad output schema diluted the work.

Fix: lean protocol + fix the 3 bugs.

05

The rules (protocol) — current vs proposed

The system prompt sets what the agent investigates. The proposal narrows four questions to one.

CURRENT investigate FOUR questions
1
Program function — what do the program genes do together?
2
Regulator mechanism — how do the perturbation regulators control this gene set?
3
Perturbation-direction consistency — does the literature agree with the observed direction?
4
Competing / contradictory explanations — actively look for the paper that disagrees.
PROPOSED investigate ONE question
1
Program function — what do the program genes + unique genes do together? (the only task)
2
Regulator mechanism — dropped as a task
3
Direction consistency — dropped as a task
4
Competing explanations — dropped as a task
Flag a contradiction only if it surfaces naturally — do not go looking for one.

Non-negotiables — kept either way

  • Retrieve first, then write — never from memory.
  • Reference only tool-returned PMIDs / DOIs — never fabricate an identifier, title, year, or quotation.
  • Do not assign the final label — the downstream synthesis does that.
  • The deterministic verifier still resolves every identifier afterward, regardless of protocol length.

The formatted output — ResearchResult

The single structured payload each session submits, validated against the Pydantic schema. Below: the schema, how the lean protocol changes each field, and a program-10 shape illustration.

 ResearchResult · submit_result tool schema
ResearchResult {
  program_id,
  queries[],
  candidate_mechanisms[ { name, summary, supporting_genes[], evidence_ids[] } ],
  claims[ {
      statement,
      supporting_genes[],
      evidence_ids[],
      context_match(direct | partial | indirect),
      status(supported | partial | unsupported)
  } ],
  evidence[ {
      evidence_id, pmid, doi, title, year, study_type, relevance_note,
      // verifier-added: resolved, registry, retracted
  } ],
  contradictions[],
  evidence_gaps[],
  agent_summary
}

→lighter — fewer questions, fewer gaps
